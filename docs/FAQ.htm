Frequently-asked Questions
This FAQ is for Fauxdacious and how to use it's extra features not found in Audacious.
1) What is Fauxdacious?
Fauxdacious is a fork of the popular Audacious Audioplayer app. which adds many new features, but which retains nearly all the interface and features of Audacious. Long-time Audacious users should find Fauxdacious a complete drop-in replacement for Audacious. However, since it is now a completely separate program, Audacious users can install and try out Fauxdacious without any risk to their existing Audacious installation or configuration. If you wish to copy your existing Audacious configuration for starters in Fauxdacious, simply: (NOTE: If you've started Fauxdacious at least once, then ~/.config/fauxdacious/ will already exist and you won't need to create it!)
cd ~/.config; mkdir fauxdacious; cp -R audacious/. fauxdacious/
2) Why Fauxdacious?
Fauxdacious has it's roots back around 2011 when I (the author) began experimenting with a few "hacks" to add some extra command-line options and tools to Audacious version 1.6 or so, which the Audacious team was not interested in including. The real forkoff came in 2015 with Audacious v3.6 when I created a video-playing option for the ffmpeg plugin. I felt that since Audacious included "visualization plugins" already, why not include one for actually displaying the video part of audio streams that contained video? After much work to add video play, I submitted the patched plugin to Audacious as a patch, but the Audacious team made it clear that they were not interested in supporting / maintaining that feature. They suggested using a video player such as VLC. I, however, really like Audacious and it's audiophileish sound-improving effects plugins and Winamp Skins interface, much preferring Audacious to VLC, and preferred having a single "go-to" media player for playing all my media. Therefore I decided to simply "fork" the code and maintain a separate Audacious branch that supported this along with it's growing list of feature patches, because I felt that other Audacious-lovers would also be interested in these new features. I've also created new logos and and changed the branding to the new name "Fauxdacious". I came up with this name by altering the name "Audacious" to have the French "Faux" like "Faux Pas" meaning "fake" or "not exactly the real thing" and liked the way it sounded! It's pronounced either "Faw-d`æ-shuhs" (rhiming with Audacious) or "Fō-d`æ-shuhs". I made the logo by using the basic "audacious" logo but adding a row of speakers instead of bubbles, a base ("F" cleff for the "F", and the slightly-rocked "X" from the default XMMS skin's logo in the lower right. I made a separate logo for the Windows version replacing the rocked "X" with a rocked "MS-Windows flag logo" that contains a swirled "X". The round icon replaces the "a" in a bubble with the aformentioned base-cleft "F". NOTE: I created this name and these logos and reserve all implied copyright rights to them as unique trademarks of "Fauxdacious"(tm). They are not to be used for any other purpose without my explicit written permission. The software code, a derived work, however is fully open-source" subject to the BSD license (the same license that Audacious uses) and you are free to use, modify and re-release your changes subject to those terms (so long as you change the name and logos).
3) How do I get and install Fauxdacious?
Download the latest source code for both Fauxdacious and the Fauxdacious Plugins from my official Github site. Then follow the instructons in the README file.

Yes, while of course I have competitive juices and am proud of my work on Fauxdacious, I owe a great debt to the Audacious project for even making this possible. My ultimate goal would be to see all of my work incorporated into mainline Audacious and for my project to simply disappear and be swallowed up by them. 6) How do I play videos?
For regular video files, ie. .avi, .mov, .mp4, .m4a, .flac, etc. Fauxdacious should now default to simply playing them just like any other audio stream. If not, simply go to [Settings].[Plugins].[Input tab].[FFmpeg Plugin], click the [Settings] button, and check the box for "Play video stream in popup window...". In short, Fauxdacious should play pretty much any video that FFmpeg's "ffplay" will play EXCEPT videos that do NOT have an audio stream (video-only streams) as audio is Fauxdacious' primary job and video is essentially a visualization. NOTE: If video seems "jittery", raise the "Video packet queue size" slightly (in [Settings].[Plugins].[Input tab].[FFmpeg Plugin].[Settings]. If video and audio are noticably out of sync, lower this value and perhaps disable some of your "Effects" plugins.
7) How do I play Youtube / Vimeo / etc. videos?
These video stream urls are not normally directly playable, but link to a temporary stream url that "times out" after a period of time and that to my knowledge, only "youtube-dl" knows how to extract. Fauxdacious CAN play them however, but it requires "helpers": a helper script written in Perl (included)! You will need to do the following (one-time setup):
Windows users: All you should need to do is go to [Settings].[Advanced] and in the box under "URL-Helper program?:" type in: "FauxdaciousUrlHelper.exe" and change the "Allow URL-helper:" dropdown from "Never" to "URLs NOT in playlists" and you should be all set!
Linux/Mac/Unix users: You must copy the script file: "FauxdaciousUrlHelper.pl" from the "contrib" directory to somewhere in your executable path ie. /usr/local/bin/. Then, depending on your Linux system, you will likely also need to install some additional Perl modules from CPAN (https://www.cpan.org), which is usually not difficult to do. The additional modules (which should be installed by the Perl package manager when you install "StreamFinder") are: "StreamFinder", "LWP::Simple", "LWP::UserAgent", and "URI::Escape". NOTE: You will also need to install either the "youtube-dl" package for your distro or download and install it yourself. "youtube-dl" is a separate program that handles the actual "downloading" of the video. In the case of Fauxdacious, the video is not actually downloaded to a file on your hard disk, but simply translated into a playable stream and metadata and a "cover-art" image is downloaded. NOTE: You will also need to go to [Settings].[Advanced], and add the full path and filename of the helper script you just copied over into the "URL-helper program?" field, and toogle the "Allow URL-helper:" dropdown to something other than "Never" ("Urls NOT in playlists" is reccomended except when you intend to load several Youtube-videos at once in a playlist, which can cause playlists with URLs to load VERY SLOWLY). Once these programs and modules are installed, Fauxdacious should automatically play Youtube and Vimeo, etc. video urls as if they were any other audio / video stream.
In summary (Windows users only need to do steps 4-5), before playing Youtube / Vimeo videos, do the following (one time):
1) Install youtube-dl.
2) Install Perl modules: "StreamFinder", "LWP::Simple", "LWP::UserAgent", and "URI::Escape".
3) Copy [source-dir]/fauxdacious/contrib/*.pl to /usr/local/bin.
4) Change the "Allow URL-helper:" box to "URLs NOT in playlists" under [Settings].[Advanced].
5) Change the "URL-Helper program?" field to: "/usr/local/bin/FauxdaciousUrlHelper.pl" ("FauxdaciousUrlHelper.exe" for Windows users) under [Settings].[Advanced].

NOTE: ALL users will need to repeat steps 4-5 once for any new Fauxdacious instances you create and wish to play videos on.

Then, to play a video:
fauxdacious https://www.youtube.com/watch?v=#######
Under the hood, Fauxdacious will create or add to a temporary file: "~/.config/fauxdacious/tmp_tag_data" which will have the actual streaming url, something like "[https://r6---sn-q4fl6n7s.googlevideo.com/videoplayback...", a title line, etc. and a comment line containing the path to the downloaded "cover-art" file, ie. Comment=file://$HOME/.config/fauxdacious/####.jpg. This file and the downloaded cover-art file will be deleted when Fauxdacious is restarted with "-D".
8) How do I record a video, ie. save it to a file for future play?
BEFORE starting playback, go to [Settings].[Plugins].[Input tab].[FFmpeg Plugin], Check the box for "Record video to file". You can also change the default filename from "/tmp/lastvideo" to whatever you like. Then, play the video all the way through but don't let Fauxdacious advance to another video entry, but stop play (An easy and handy way to do this is drag your Youtube video to the bottom of your playlist and turn [Repeat] OFF), then go back to [Settings].[Plugins].[Input tab].[FFmpeg Plugin], uncheck the "Record" box (to prevent overwriting), then you have the video in the file you chose (default "/tmp/lastvideo"), which you'll likely want to move and rename to something with a proper extension, ie. ".m4a" usually works. After moving the saved video file to wherever you want it to permanently reside, if you want to save the metadata and cover-art file, you will need to edit both ~/.config/fauxdacious/tmp_tag_data and either ~/.config/fauxdacious/user_tag_data OR <path to saved video file>/user_tag_data.tag, and MOVE the lines starting with [https://r6---sn-q4fl6n7s.googlevideo...] and ending with a blank line from the "tmp_tag_data" file to one of the other file you opened. Then change the "https://r6---sn-q4fl6n7s.googlevideo..." part to "file://<path to your saved video file>". Save both files. You can also move the cover-art image to there as well and edit the "Comment=" line to it's new location. This saves your tag data and cover-art image for the new video file.
9) How do I play DVDs?
IF playing commercial (encrypted) DVDs (AND it is LEGAL for you to do so in your jurisdiction!), you'll will need to install "libdvdcss". NOTE: Windows users will need to obtain and copy libdvdcss.dll to the Fauxdacious bin directory (the same directory fauxdacious.exe and libdvdread.dll reside in) each time you install or update Fauxdacious (Sorry, I can't legally include this in the installer)! For everyone else, if you are already doing this in VLC or some other open-source video player on your machine, then you probably already have this installed. Then, either start Fauxdacious with:
fauxdacious dvd://
or open Fauxdacous the usual way and go to [Services] menu and select either "Add DVD" or "Play DVD". If you have trouble playing a specific DVD, you can adjust the settings in [Settings].[Plugins].[Input tab].[DVD Plugin]. If your dvd device is unusual, ie. something other than "/dev/dvd", change this box. Make sure the "Play video stream in popup window..." box is checked (unless you just want audio). You'll probably want the "Highlight menu buttons" box checked. To avoid clutter in your playlist, most users will want "Only Title Track in Playlist" checked. If the DVD menu doesn't seem to work (some DVDs just don't), you may want to also either check the "Only 1st track in Playlist" and/or the "Only Longest Track in Playlist" box, which is usually just the main feature movie. If none of the "Only..." boxes are checked, each "track" file on the DVD will appear in the playlist (giving it a cluttered appearance, and is not usually very useful). "Continue at end of menus" and "Skip Menus" should only be checked if a DVD's menus don't work otherwise, as this can cause some DVDs to crash Fauxdacious. It's also helpful to have one or the other checked (experiment) if just playing audio (in which case there's no way for the user to interact with the DVD menus). Check the box for "Allow Custom Tag-files" to allow usage of custom tag files which you can either create manually and/or use the save Song Info box to create and save metadata for the DVD tracks. The custom tag file for a DVD will be "~/.config/fauxdacious[_instancename]/DVD_TITLE.tag". NOTE: This is also true for audio-CDs ("~/.config/fauxdacious[_instancename]/DISK-ID.tag"), except audio-CDs can also fetch metadata for individual tracks from freedb.com, or musicbrainz.com using the cover-art "helper" script (mentioned next paragraph below) and create the custom tag file (along with downloading any cover-art image) for you the first time you play the CD. To have Fauxdacious attempt to fetch the DVD's "cover" image from the internet, go to [Settings].[Advanced] and add the following to the "CD/DVD Cover Art-helper program?:" field:
/usr/local/bin/FauxdaciousCoverArtHelper.pl NOTE: For Windows users, this will just be: FauxdaciousCoverArtHelper.exe (it's already in the path)!
Linux/Mac/Unix users: Make sure the script file "FauxdaciousCoverArtHelper.pl" is in /usr/local/bin, otherwise specify it's location and make sure it's executable). NOTE: The cover-art helper script may not be able to fetch a proper art image for all DVDs, in which case, it will not have one.
10) How do I use custom equalizer and / or effects plugin presets for specific songs, directories, disks, etc.?
NOTE, this is NOW available in the Qt interfaces too (as of v3.10)! To set custom equalizer presets for a specific song, stream, or CD / DVD disk, first start playing it. Then open up the Equalizer window and make sure both the [Auto] and [Equalizer] buttons (or checkboxes, depending on interface) are selected. Then press the [Preset(s)] button. A dialog window will pop up. There are three additional checkboxes on this window: "Use entry's local dir..." specifies try to save the presets in the same directory as the song file / cue-sheet file is in (only applicable if the song is a local file, not a disk or url). Otherwise, the custom presets will be saved in the ~/.config/fauxdacious[_instancename]/ directory. The "Exclude arg. lists from urls..." box only applies to URLs. Checking this removes the "?..." crap off of the URL before naming the custom preset file. This is useful if one doesn't want to create long nasty preset file names for every possible URL on the site, but can limit customization. Experiment as ymmv. Uncheck the "Save Effects..." checkbox if you only want to save equalizer presets for the song and not the current list of Effects plugins being used. Next, choose [Export], then [Preset File] from the menu. A window should pop up with the proper name for the custom preset file filled in and the current directory set to either the local file's path or ~/.config/fauxdacious[_instancename]/ directory (depending on whether it's a local file and whether the "Use entry's local directory..." box is checked. [Save] the preset file. If instead, the filename says "<name>.preset", then either the song is NOT actually PLAYING, you selected "EQF file" instead of "Preset file", or Fauxdacious was unable to determine a valid name, ie. from a really ugly URL. In this case try cancelling the file dialog window, toggle the "Exclude arg. lists from URLs" checkbox, and try the Export again. Once saved and activated, the [Preset(s)] button should light up indicating that Fauxdacious is using a custom preset file for this song entry. Now, adjust the equalizer how you want it for this song, and check / uncheck any Effects plugins you want on / off for it (if "Save Effects..." checked), then repeat the [Save] step to re-save the custom preset file with new values as the custom presets for this entry. When the entry stops playing and Fauxdacious advances to the next song entry, the equalizer settings and Effects plugins selections should return automatically to the defaults (what they were before you saved the custom file the first time and adjusted them). The [Preset(s)] button's light / indicator should go off as well. NOTE: The song entry must be actually PLAYING when all this is being done!
11) How to set custom presets for an entire subdirectory of song files (the way XMMS can) / OR use a custom directory preset file that was created in XMMS?
"dir_default.preset" is the name XMMS (and Fauxdacious) uses to create custom directory presets. If you haven't used XMMS to create them or don't want to use those, you can change this name to whatever you want by editing your ~/.config/fauxdacious[_instancename].config file with your favorite text editor and change "dir_default.preset" in the following line:
eqpreset_dir_default_file=dir_default.preset
to some other preset file name, OR remove the line if you don't want to allow for directory-based presets at all as an option. Then with a song playing in the directory for which you wish to set a custom directory preset for, follow the same procedure outlined above for saving the custom preset file, but make sure the BOTH the "Use entry's local directory..." and "Save as Directory-wide..." boxes are checked, then when preparing to save the file, the song's filename part of the preset file name ("<song-filename>.preset") should read "dir_default.preset" or whatever you specified above, for "eqpreset_dir_default_file", and save the preset file. This preset file will be used for equalizer and optionally Effects plugin settings for every song in this directory (but NOT subdirectories) EXCEPT for any that have a custom song-specific preset file saved for them. Fauxdacious searches for custom preset files in the following order: 1) <path-to-song-file>/<song-filename>.preset 2) ~/.config/fauxdacious[_instancename]/<song-filename>.preset 3) <path-to-song-file>/dir_default.preset (if eqpreset_dir_default_file set) Only #2 is used for non-local file entries, ie. URLs, CDs, and DVDs. For CDs, the <song-filename> is the CD's disk-ID, for DVD's it's the DVD Title.
12) How do I pipe media, cue-sheets, playlists, etc. into Fauxdacious from stdin?
In addition to most media, Fauxdacious allows cue-sheets, playlists, and plain old text lists of entries to be piped into Fauxdacious via stdin. Fauxdacious originally did this but required the user to provide an extension as a "hint" so that it could determine what type of media was being piped in since probing to determine media type would read stdin and there was no way to rewind it to play. For example:
cat mysong.mp3 | fauxdacious -.mp3
Audacious later added this ability (at my urging) along with some black buffering magic that allowed Audacious to first probe stdin and determine the media type, then play, thus not requiring or using the extension. Fauxdacious now includes these improvements to Audacious so the extension hint is no longer required for most media (but is still allowed, see next paragraph and #1 below), ie.:
cat mysong.mp3 | fauxdacious -
However, Fauxdacious permits non-media to also be piped in and the underlying Audacious code does not support "probing" for playlists and cue-sheets thus still REQUIRING the extension-hint for these types of input. Therefore, we've retained the extension-hint (now optional for most media) for consistancy, but also to allow Fauxdacious to skip the overhead of probing when the media type is known by extension! There are three exceptions:
1) Certain media files have been known to FAIL when piped in and probed thus still necessitating including the extension hint to get them to play via stdin. A handful will work the new way but not the old too.
2) If no extension-hint is given and Fauxdacious fails to probe and find a suitable media plugin for the input, it will "fall back" by trying to treat the input stream as a text list (m3u) playlist and, if successful, load the playlist from stdin without the extension hint. Cue-sheets and all other playlist types require the extension hint to load from stdin.
3) A special case for strictly text list input from stdin is provided by specifying a double-dash ("--") for stdin whicn, instead of loading the incoming list of entries as an m3u "playlist" will treat them as if entered on the command-line (much earlier in the program startup). This way is necessary if attempting to load additional playlist entries to an already-running instance of Fauxdacious (since the additional files must be loaded BEFORE the second command hands off control to the running instance). Example:
ls -d A*.mp3 | fauxdacious -Dc - &
ls -d B*.mp3 | fauxdacious -e --
The second command will add (enqueue) all mp3 files in the current directory starting with "B" to the bottom of the playlist in the already-running instance of Fauxdacious started by the first command, then the 2nd command will exit. NOTE, it's not PLAYING the songs themselves here through stdin, but the playlist (list of entries), thus song-playing is seekable and replayable.
13) How do I play media to stdout for post-processing instead of through my speakers?
There are two ways of doing this: 1) The FileWriter plugin has an additional option to write it's output to stdout instead of to a named file. This can be used by either selecting the FileWriter plugin or by using the "Record" option to dub (tee) output to FileWriter. In either case, the FileWriter plugin must be set to write to stdout and Fauxdacious must be invoked from the command line with a target for stdout, ie:
fauxdacious [stuff-to-play] >/tmp/outputfile.mp3
fauxdacious [stuff-to-play] | mplayer -
Otherwise, stdout will dump tons of "garbage" right out to your terminal (pretty useless)!
The second (new, easier) way is to add the "-o" or "--out=<ext>" command-line option to Fauxdacious when starting. "<ext>" can be either "wav", "mp3", "ogg", or "flac". If just "-o" is specified, it is the same as "--out=wav".
This method automatically directs Fauxdacious to send all output to stdout and in the format selected without having to specify "stdout" in the FileWriter options or reset anything when done. Fauxdacious will remember which audio output plugin ("Alsa", "Pulseaudio", etc.) you were using and what options you had FileWriter set to for recording / dubbing, and reset everything back when done. Here's a true Unix piping example to convert a bunch of files to a single .wav file in "headless" mode (no GUI). This will run Fauxdacious to do the conversion, then exit and restore everything to normal:
ls -d * | fauxdacious -DcHo - >/tmp/newfile.wav
If using the first way (FileWriter / Record), there are two additional plugin configuration options under stdout: "close after song chg." and "close when recording stopped.". They are useful when piping directly out to a stored file (as opposed to piping to another program downstream). The first says to "close" (send an EOF, etc.) to stdout when the song changes (either by advancing to the next song entry or when title changes in a radio-station URL). Use this when recording (dubbing) a song streaming stations, etc. as it will ensure a properly created song file of the LAST song recorded when recording is stopped (but any prev. song written out will be lost). Unchecking ensures that everything written to stdout during the recording is included in the file produced by stdout as a single file. However, stopping recording and starting a new recording will overwrite stdout with whatever's recorded by the second recording! In order to create a continuous recording by stringing together everything from each recording (until Fauxdacious itself is stopped), uncheck the "close when recording stopped" box. Also, uncheck both boxes if piping to another downstream program instead of a stored file. Using the "-o/--out=<ext>" option will automatically toggle both these boxes off while piping, then reset them to their prev. values when Fauxdacious exits. The tradeoff here though is that if both boxes are unchecked and you're piping to a stored file, the tag metadata from the first song is likely used for the file.
14) How do I get Fauxdacious to just mute but continue streaming whenever paused - I want to mute commercials in my streaming radio station and either toggle over to another Fauxdacious instance where I have my music ready to play a couple songs -or- play a Facebook video in my web-browser during the break? If I "pause" the station playing commercials, it'll resume at the BEGINNING of the commercials.
Fauxdacious adds the "-P" option just for this annoying situation! If Fauxdacious is started with "-P", pressing the [Pause] button will silence whatever's playing in that instance without changing or affecting your volume controls! Pressing [Pause] again will restore playing volume at that point in play and the station will never know that you "skipped" all their commercials!
You can also now toggle this feature dynamically either by going to [Settings].[Playlist] and toggling the checkbox labeled "Pause mutes instead of pausing" OR using fauxdtool:
fauxdtool [instance instancename] --playback-pausemute [0|1]

"1", "T[rue]", "t[rue]", "Y[es]", "y[es]", or no argument turns Pausemute ON.
"0", "F[alse]", "f[alse]", "N[o]", or "n[o]" turns it OFF.
To check the current status of PauseMute using fauxdtool in a batch file, do:
fauxdtool playback-doespausemute
if [ "X$?" = "X1" ]; then
    echo "pause will just mute"
else
    echo "pause will pause"
fi
15) How do I save tag metadata (ie. song title, artist, cover-art, etc.) using the "Song Info." dialog or manually for online streams, CDs, DVDs, and files in audio-formats that do not support the metadata tag format used by Audacious?
Unlike Audacious, Fauxdacious can still save your custom tag data associated with song entries whether they support it or not. Fauxdacious first tries to save the tag data in the song file itself, and failing that, can save it to a file, and read the metadata from that file whenever that song entry is played, as if the tag data were saved in the song entry file itself (NOTE: These tags will not be present if playing in Audacious or another media player though)! For most entries, the "Song Info." dialog will save the tag data in the common tag file: "~/.config/fauxdacious[_instancename]/user_tag_data". For disks (CDs and DVDs), it will be saved in a separate custom tag file "~/.config/fauxdacious[_instancename]/DISK-ID.tag. For streams downloaded via youtube-dl through the helper-script (which expire and thus are considered "one-off" plays), a separate "tmp_tag_data" file is used. CDs and DVDs also store tag data there while they are playing. When playing an entry, Fauxdacious will look for tag data to fill in the metadata fields first in "~/.config/fauxdacious[_instancename]/tmp_tag_data", then (if song is a local file) in user_tag_data.tag in the directory the song file is in, then in "~/.config/fauxdacious[_instancename]/user_tag_data", then (if song is a CD/DVD track, in "~/.config/fauxdacious[_instancename]/DISK-ID.tag", and last, any tag data stored in the audio file itself. These are all editable text files. One of the argument fields for each tag is "Precedence" which can be set to "DEFAULT", "OVERRIDE", or "ONLY", which while processing the above tag sources, will either use as default, override any value from a previous source found, or ONLY use THAT value. Another special field is the "Comment" field, which can be any text comment, but if set to the full path of a valid image file name, will be used as the "cover art image" for the song entry! To turn these features on, go to [Settings].[Song Info] and check the "Save unsavable metadata to file" box. If unchecked, then Fauxdacious will be limited to handling most metadata the same way Audacious does. NOTE: See also the "Allow Custom Tag-files" options under [Settings].[Plugins].[Audio Cd/DVD plugins] configurations for allowing these custom tag files for individual disks.
16) When I start up Audacious from the command line with a list of stuff to play, Audacious creates a new "playlist", which is very annoying since I use the WinAmp Skins interface and don't see the previous ones it created, which gets annoying since I end up with a bunch of old, one-off playlists before I realise it! How do I start up Fauxdacious with just a single playlist, or even just with the files I specified on the command line?
To start up Fauxdacious with just a single playlist and discard any other existing ones, start with the "-D" (Delete playlists) option. To clear the current playlist of any existing entries, start with the "-c" (Clear playlist) option. To start Fauxdacious with a completely clean slate, start up with both "-D" and "-c" (or "-Dc").
17) How do I start up Fauxdacious with the equalizer turned OFF?
Start up Fauxdacious with the "-z" option. Fauxdacious will remember whether it was on before you started up and restore that when exiting.
18) How do I start up Fauxdacious with "Recording" (dubbing) turned ON?
Start up Fauxdacious with the "-R" option. NOTE: This will be ignored if the "-o/--out=<ext>" option is also specified!
19) How do I start up a second "instance" of Fauxdacious or one that uses a configuration different than my normal "default" configuration?
There are three different ways to start up multiple or different Fauxdacious instances:
1) fauxdacious --new=instancename
This is the preferred way as it allows you to "name" your different configurations (configuration directories) in order to have Fauxdacious set up multiple ways for different scenarios. "instancename" can be any string of letters and digits up to 40 characters. To create a completely "new" instance that you've never used before, just specify any name you choose, and Fauxdacious will create a brand new "~/.config/fauxdacious_<instancename>/" directory along with a factory-default "config" file. If the directory already exists (Fauxdacious has been started before with this instance name), that configuration directory will be used. This way, unlike in Audacious, you can have more than nine different configurations named to where you can remember what they're for!
2) fauxdacious -# (or fauxdacious --new=#)
This is the "Audacious way" of creating new instances is by specifying a single digit, which Fauxdacious will use as the instance "name". NOTE: Fauxdacious creates a corresponding "~/.config/fauxdacious_#/" if it does not already exist, just as if you'd given the instance an alphanumeric name. NOTE the underscore ("_") instead of a dash ("-") in the directory name. Audacious permits -1 .. -9, but -1 is the same as the default instance. In Fauxdacious, -0 refers to the default instance and -1 .. -9 are separate instance names. I can't remember why we had to change the "-" to "_" though.
3) fauxdacious -n
This creates a new, separate running (but unnamed) Fauxdacious instance using the default config directory ("~/.config/fauxdacious/"). I've never found this to be of much use, but for each multicharacter argument that takes a value, the code requires a corresponding single-character version with no arguments.
20) How do I control different running Fauxdacious instances using the Fauxdtool?
To control the default instance:
fauxdtool [--command...]
To control a different instance name (--new=<instancename>):
fauxdtool instance <instancename> [--command...]
To control a numeric instance (created the Audacious way with -#):
fauxdtool -# [--command...]
One can not access nor control an unnamed instance created with "-n" with the Fauxdtool though.
21) How do I get Fauxdacious to randomly change "skins" when the playlist entry changes (like the old XMMS "Random Skin on Play" option)?
Go to [Settings].[Appearance], switch Fauxdacious's "Interface" to the "Winamp Classic Interface", then check the box at the bottom of the window labeled "Random skin on play". Fauxdacious will randomly change it's "skin" each time the playlist advances. NOTE: Streaming radio stations that change song titles between songs does NOT cause the skin to change, however. I may look into adding that in the future if there is any demand for it.
22) How do I use local lyrics files in the Fauxdacious version of the lyricwiki plugin, as Audacious does not (currently) support this?
When the lyricwiki plugin is active and Fauxdacious successfully fetches lyrics from lyricwiki, there's a new button (right-click menu item) labeled [Save Locally]. If valid lyrics from lyricwiki are found and displayed, you may select this option and a lyric file will be created matching the name of the file being played OR song title (for files) saved in the same directory as the song entry / cuesheet file. For example if playing "myfavoritesong.mp3", the lyric file will be created as "myfavoritesong.lrc". If playing a song not in a local file, ie. from a stream URL or CD, the file "~/.config/fauxdacious[_instancename]/lyrics/Song Title.lrc" (ie. if the song's title is "Song Title") will be created if you save the lyrics locally. An example would be recording a streaming song whilst playing, then manually renaming the recorded song file to "myfavoritesong.mp3" and optionally renaming the lyric file to "myfavoritesong.lrc", then and moving both to where you keep your music. NOTE: The recorded song file will be the newest file in /tmp/ and will have a different name from the title (see [Settings].[Audio].Recording Settings.[Settings] and/or the FileWriter plugin's settings). NOTE2: The [Save Locally] option will be greyed out whenever Fauxdacious has successfully fetched the lyrics from an existing local lyrics (.lrc) file for the current song, or if no valid lyrics were found on lyricwiki.
When playing a song, ie. "myfavoritesong.mp3", Fauxdacious will first search for a lyric file ("myfavoritesong.lrc") in the same directory the song is in, then for "myfavoritesong.lrc" in ~/.config/fauxdacious[_instancename]/lyrics/. If neither are found, it will search both locations for an lrc file matching the song's title, ie. "Song Title.lrc". Failing that, Fauxdacious will seek to pull lyrics from lyrics.fandom.com just like Audacious.
You can also manually create .lrc (text) files for the tracks of your favorite CDs as well! You will need the CD's numeric disk-ID (Fauxdacious can fetch the cover art, etc. for commercial CDs and will store it as ~/.config/fauxdacious[_instancename]/[disk-ID].jpg, for example. First, manually create the directory: "~/.config/fauxdacious[_instancename]/lyrics/[disk-ID]/". Then create the lyric files for each track for which you have lyrics, naming them: "track_#.lrc" there ("#" is the track number. The first track on the disk would be track_1.lrc). Then, when you play the CD, the lyrics will be fetched for each track as it plays. TIP: If Fauxdacious is able to obtain the lyrics for a given track from lyricwiki, you can press the [Save Local] button, which will save the lyrics for that track under ~/.config/fauxdacious[_instancename]/lyrics/[Song Title].lrc, which you can either leave in place OR move and rename it to the corresponding track file: "~/.config/fauxdacious[_instancename]/lyrics/[disk-ID]/track_#.lrc".

25) How do I reverse the sort order when clicking on the playlist column-headers in either of the "non-Winamp-skins" interfaces? Audacious only sorts them in ascending order.
Click on the same column header a second time and the sort will be reversed and the little sort arrow (on the Qt-side) will reverse direction (Fauxdacious only)! NOTE: The sort arrow is not done when sorting via the sort menu, unfortunately. Also note that the "Entry#" column is simply a "line#" column and does not sort (stays in ascending order 1,2,...)! You can also sort by bitrate in Fauxdacious.
26) What is Fauxdacious latest version#, and do you follow Audacious versioning?
We are currently at version# 4.0-beta3. Yes, I try to stay pretty close to theirs, but even though the "official" version# may often be a bit behind Audacious, it usually contains many, if not most of their latest version#'s GIT commits (after they settle down and I get a chance to test, and often combine them). 27) Audacious display of Shoutcast streams' titles is sometimes ugly and, at best displays both the artist and title in the Title field, but I want title in the Title field and artist in the Artist field (like the rest of my local music file collection), and put the stream name in the Album field instead. How can I fix this?
Fauxdacious already cleans up "ugly" Shoutcast stream titles to where the title is consistantly displayed as "artist - title" the stream name is put in the Artist field, and the station's URL shows in the Album field. To change this, there's a new checkbox option under [Settings].[Playlists] labeled "Split Artist - Title in streams". Check this box and the Title and Artist fields will be parsed out separately into their respective fields (like normal song files with metadata), and the stream name will be placed in the Album field instead. If the stream also sets the Album field to the actual album name of the song being played, it will be prepended to the stream name in the Album field as: "album name - stream name". NOTE: The URL helper that fetches Shoutcast stream urls places the station's website (in Reciva, Tunein, IHeart, etc.) in the Album field. This will be overwritten by the Album info described above when a stream is played, when this checkbox is checked. This was done because I believe the stream and album name is more useful being displayed than the station's original website URL. NOTE2: aac (libfaad) (v2.8.8-3.1+) seems to be busted (on Buster) and fails to play many (audio/aacp) streams so that for the time being, we have the ffaudio (ffmpeg) plugin grabbing these, but ffaudio doesn't yet dynamically update the metadata, so many streams may not have metadata (for the time being).
